# 🧠 Computer Architecture – MIPS (MARS)

This repository documents my journey learning low-level programming as part of a Computer Architecture course.

The focus is on understanding how software interacts with hardware by working directly with assembly language, specifically MIPS, using the MARS (MIPS Assembler and Runtime Simulator) environment.

---

## 📌 Purpose

This repo is not just a collection of code — it’s a structured learning log.

The goal is to:
- Build a solid mental model of how programs execute at the hardware level
- Understand memory, registers, control flow, and instruction execution
- Practice translating high-level logic into assembly instructions
- Document insights, patterns, and mistakes along the way

---

## 🧠 What I'm Learning

This repository covers key Computer Architecture concepts such as:

- Instruction execution cycle (fetch → decode → execute)
- Registers and memory hierarchy
- Control flow (branches, jumps)
- Stack usage and function calls
- Data representation and addressing
- Performance considerations at the hardware level

## 📈 Approach

I treat this repository as a thinking tool, not just storage.

For each exercise:
- Understand the problem in high-level terms first
- Break it down into step-by-step operations
- Map those operations to MIPS instructions
- Reflect on what actually happens at the hardware level

## 📚 Why This Matters

High-level languages abstract away the machine.

Assembly forces you to confront:
- How memory is actually used
- How instructions are executed step-by-step
- Why performance depends on access patterns and control flow

This is where you stop guessing and start understanding.
